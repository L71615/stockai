## 一、现状总结

| 维度 | 现状 |
|------|------|
| **功能** | 12 页面、29 因子、5 AI Agent、多数据源 |
| **工程质量** | 测试 ~20%、零 CI/CD、无 ORM、裸 except 25+ 处 |
| **安全** | D+ — JWT 默认密钥、CORS 全开、AI Key 明文存储 |
| **架构** | C+ — SQLite 单机、无索引、无备份、认证未完成 |
| **产品定位** | 模糊 — 作品集定位 vs 商业产品定位矛盾 |

## 五、明确产品定位

> **核心目标**：面向面试官展示技术力（一直是主要目标）
> **探索方向**：面向学生群体（金融比赛/投资教育/信息壁垒）— 设为设想，暂不推进

**面向面试官展示的核心技术力**：

| 技术维度 | 项目中对应的实现 |
|---|---|
| 前后端分离架构 | Next.js + FastAPI 解耦 |
| AI 多供应商集成 | MiniMax/DeepSeek/Claude 统一调度 + 5 Agent 交叉验证 |
| 量化因子体系 | 29 因子 IC 加权 + 多数据源融合 |
| 工程规范 | 安全加固（JWT/CORS/加密/限流）+ CI/CD + 测试覆盖 |
| 数据工程 | 多源行情兜底链路 + SQLite WAL + 进程内缓存优化 |
| 前端工程 | SWR 缓存 + 响应式设计 + OKLCH 设计系统 |

**学生市场探索方向**（暂不推进，设为设想）：
- 比赛推送（金融比赛信息聚合）
- 券商平台对接（模拟交易/开户数据）
- 投资教育引导（抹平信息壁垒）
