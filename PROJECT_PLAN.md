# StockAI — 项目规划

## 项目概述

A 股持仓追踪 + AI 分析助手。支持网页版（手机 App 后续）。

---

## 一、当前技术栈

| 层面 | 方案 |
|------|------|
| 前端 | HTML5 + CSS3 + Vanilla JS（深色主题） |
| 后端 | Python FastAPI + uvicorn |
| 数据库 | SQLite（WAL 模式） |
| 股市数据 | 东方财富 API（push2.eastmoney.com，subprocess curl 绕过 TLS 指纹） |
| AI 模型 | MiniMax（OpenAI 兼容协议，默认 MiniMax-M2.7），支持 Claude / OpenAI / 自定义 |
| 部署 | 本地开发，Docker 待定 |

---

## 二、已实现

- [x] **前端 8 页面** — 持仓/自选股/大盘指数/AI 助手/Skills 商店/交易记录/设置/登录
- [x] **共享 CSS/JS** — 深色主题、侧边栏导航、通用组件

## 三、功能路线图（17 项）

### 第一批：基础数据真实化
- [x] 1. 清空假数据 — 数据库持仓/自选归零
- [x] 2. 持仓页表单 — 前端添加表单（股票代码/名称/成本/数量/市场）
- [x] 3. 实时盈亏 — 持仓页 KPI 卡片（总市值/成本/今日盈亏）+ 表格每行盈亏

### 第二批：Agent + 新闻
- [x] 4. Agent 工坊页 — 替换 Skills 商店页，创建/编辑/删除 Agent（名称/提示词/工具/记忆）
- [x] 5. AI 助手改造 — Agent 选择器 + 新闻列表 + 快捷指令按钮
- [x] 6. MD 记忆系统 — 每个 Agent 独立 .md 记忆文件，跨会话持久化，启动时加载
- [x] 7. 新闻爬取+匹配 — 爬东方财富/新浪，按行业标签匹配用户持仓

### 第三批：定投 + 复盘
- [x] 8. 定投计划 — 标记持仓为定投，周期/金额/下次扣款日
- [x] 9. 定投备忘录 — AI 在扣款日前生成提醒，显示在持仓页
- [x] 10. AI 复盘 — 读取交易记录，分析盈亏原因，生成复盘报告

### 第四批：进阶
- [x] 11. 持仓分散度 — 行业/市场占比图表，集中风险提醒
- [x] 12. 技术指标 — KDJ/MACD/RSI/均线，AI 解读信号
- [x] 13. 价格预警 — 涨跌幅超阈值通知（数据库表已建）
- [x] 14. 股息记录 — 分红除权追踪，含股息的实际年化收益

### 第五批：扩展
- [x] 15. 多组合管理 — 长线/短线/实验仓分开统计
- [x] 16. 月度/季度复盘报告 — 自动生成交易总结
- [x] 17. 同行对比 — 持仓 vs 同行业均值表现（大盘指数对比）

---

## 四、用户偏好

- AI 供应商：MiniMax，starter 年度套餐，默认模型 MiniMax-M2.7
- 模型名用文本输入框，不硬编码下拉选项
- 不展示虚假/演示数据，所有数据用户自己添加
- 定投为主，长期持有策略
- 只提方案不主动实施，等用户确认后再执行
